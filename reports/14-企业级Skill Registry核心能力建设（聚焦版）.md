# 企业级 Skill Registry 核心能力建设（聚焦版）

## 一、外部 Skill 社区 vs 企业 Skill Registry

### 1.1 外部 Skill 社区（skill.sh 等）定位

**定位**：Skill **发现与分享平台**

```
┌─────────────────────────────────────────┐
│         外部 Skill 社区                   │
│  (skill.sh, GitHub, 腾讯 SkillHub...)    │
├─────────────────────────────────────────┤
│  • Skill 发现与分享                      │
│  • 个人开发者使用                        │
│  • 无企业级治理                          │
│  • 无权限控制                            │
│  • 无审计日志                            │
└─────────────────────────────────────────┘
```

### 1.2 存在的问题

| 问题 | 描述 | 企业影响 |
|------|------|---------|
| **无安全审核** | 任何人都可以上传 Skill | 🔴 恶意代码风险 |
| **无权限控制** | 公开访问，无鉴权 | 🔴 数据泄露风险 |
| **无审计日志** | 无使用记录 | 🔴 无法追溯问题 |
| **无版本治理** | 版本混乱 | 🔴 生产环境不稳定 |
| **无 SLA 保障** | 无可用性承诺 | 🔴 生产不可用 |
| **无合规检查** | 无合规验证 | 🔴 法律风险 |

### 1.3 与 Skill Registry 的关系

```
┌─────────────────────────────────────────────────────────┐
│              Skill 生态全景                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  外部 Skill 社区           企业 Skill Registry           │
│  (skill.sh 等)            (Matrix)                     │
│  ┌─────────────┐          ┌─────────────┐              │
│  │ Skill 发现   │  ────→   │ 安全审核     │              │
│  │ Skill 分享   │  导入    │ 权限管理     │              │
│  │ 公开访问    │          │ 版本治理     │              │
│  │ 无治理      │          │ 审计日志     │              │
│  └─────────────┘          └─────────────┘              │
│       ↓                        ↓                        │
│  个人/测试使用              企业生产使用                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**关系定义**：
- **外部社区** = Skill **来源池**（发现 + 分享）
- **Skill Registry** = Skill **治理平台**（审核 + 管理 + 分发）

**企业工作流**：
```
1. 从 skill.sh 发现 Skill
2. 导入到企业 Skill Registry
3. 安全审核 + 合规检查
4. 版本管理 + 权限配置
5. 分发给企业内部 Agent 使用
6. 监控使用 + 审计日志
```

## 二、企业级核心需求排序

### 2.1 需求优先级（P0-P3）

| 优先级 | 需求 | 为什么重要 | 外部社区 | Skill Registry |
|--------|------|-----------|---------|---------------|
| **P0** | **安全审核** | 防止恶意代码/数据泄露 | ❌ 无 | ✅ 核心 |
| **P0** | **权限管理** | 控制谁能访问/使用 | ❌ 无 | ✅ 核心 |
| **P1** | **版本管理** | 生产环境稳定性 | ⚠️ 弱 | ✅ 核心 |
| **P1** | **分发控制** | 确保正确 Agent 使用 | ❌ 无 | ✅ 核心 |
| **P2** | **智能检索** | 快速找到合适 Skill | ⚠️ 基础 | ✅ 增强 |
| **P3** | **审计日志** | 问题追溯/合规 | ❌ 无 | ✅ 增强 |

---

### 2.2 P0：安全审核（最重要）

**企业痛点**：
```
从 skill.sh 下载的 Skill，直接用于生产：
    ↓
可能包含：
- 恶意代码（窃取数据）
- 安全漏洞（被攻击）
- 敏感信息泄露（API Key/密码）
- 合规问题（版权/许可证）
    ↓
企业无法承担的风险
```

**核心能力**：
| 审核维度 | 实现方式 | 优先级 |
|---------|---------|--------|
| **敏感信息检测** | 正则 + NER 模型 | P0 |
| **恶意代码检测** | 静态分析 (Semgrep) | P0 |
| **漏洞扫描** | 依赖检查 (Snyk) | P0 |
| **合规检查** | 许可证验证 | P1 |
| **质量评估** | 大模型评分 | P2 |

**工作流**：
```
Skill 导入 → 敏感信息检测 → 恶意代码检测 → 漏洞扫描
    ↓
通过 → 进入候选池
    ↓
拒绝 → 通知上传者修复
```

---

### 2.3 P0：权限管理

**企业痛点**：
```
问题 1：谁可以访问 Skill？
- 公开？内部？特定团队？

问题 2：谁可以使用 Skill？
- 所有 Agent？特定 Agent？

问题 3：谁可以修改 Skill？
- 任何人？仅管理员？
```

**核心能力**：
| 权限类型 | 描述 | 实现 |
|---------|------|------|
| **访问权限** | 谁能看到 Skill | RBAC（角色权限） |
| **使用权限** | 谁能调用 Skill | Agent 白名单 |
| **修改权限** | 谁能编辑 Skill | 所有者 + 管理员 |
| **分发权限** | 谁能发布 Skill | 审核委员会 |

**权限模型**：
```
企业
├── 团队 A
│   ├── Skill 1（团队内共享）
│   └── Skill 2（仅特定 Agent）
├── 团队 B
│   └── Skill 3（全公司共享）
└── 公共库
    └── Skill 4（所有人可见）
```

---

### 2.4 P1：版本管理

**企业痛点**：
```
问题 1：生产环境用哪个版本？
- v1.0.0 稳定，但功能旧
- v2.0.0 功能新，但有 bug

问题 2：如何灰度发布？
- 先 10% Agent 使用
- 没问题再全量

问题 3：如何回滚？
- v2.0.0 有问题，快速回滚到 v1.0.0
```

**核心能力**：
| 能力 | 描述 | 外部社区 | Skill Registry |
|------|------|---------|---------------|
| **多版本并行** | DRAFT/GRAY/FORMAL | ❌ | ✅ |
| **灰度发布** | 按比例/按 Agent 分发 | ❌ | ✅ |
| **版本回滚** | 快速恢复到稳定版本 | ❌ | ✅ |
| **依赖管理** | Skill 之间的版本兼容 | ❌ | ✅ |

**版本状态机**：
```
DRAFT → 审核 → GRAY → 测试 → FORMAL → 发布
              ↓                    ↓
           拒绝                  下架
```

---

### 2.5 P1：分发控制

**企业痛点**：
```
问题 1：Skill 如何分发给 Agent？
- 手动配置？自动发现？

问题 2：如何确保 Agent 使用正确版本？
- 固定版本？动态更新？

问题 3：如何监控使用情况？
- 哪个 Agent 用了哪个 Skill？
- 使用频率？性能如何？
```

**核心能力**：
| 能力 | 描述 | 实现 |
|------|------|------|
| **Agent 绑定** | Skill 与 Agent 关联 | 配置管理 |
| **版本锁定** | 固定 Skill 版本 | 版本约束 |
| **自动更新** | 新版本自动推送 | 订阅机制 |
| **使用监控** | 统计使用情况 | 指标采集 |

**分发模型**：
```
Skill Registry
    ↓
Agent A → Skill v1.0.0 (锁定)
Agent B → Skill v1.0.0 (锁定)
Agent C → Skill v2.0.0 (灰度)
    ↓
监控使用 → 发现问题 → 快速回滚
```

---

### 2.6 P2：智能检索

**企业痛点**：
```
问题 1：如何快速找到合适的 Skill？
- 关键词搜索不够准确

问题 2：如何发现相似的 Skill？
- 避免重复造轮子

问题 3：如何评估 Skill 质量？
- 使用频率？用户评分？
```

**核心能力**：
| 能力 | 描述 | 优先级 |
|------|------|--------|
| **关键词搜索** | 基础搜索 | P1 |
| **语义搜索** | 向量化检索 | P2 |
| **标签过滤** | 按标签筛选 | P1 |
| **质量排序** | 按使用频率/评分 | P2 |

## 五、总结

### 5.1 外部社区 vs Skill Registry

| 维度 | 外部社区 | Skill Registry |
|------|---------|---------------|
| **定位** | Skill 发现与分享 | Skill 治理与管理 |
| **目标用户** | 个人开发者 | 企业 |
| **核心能力** | 上传 + 下载 | 安全 + 权限 + 版本 + 分发 |
| **关系** | **来源池** | **治理平台** |

### 5.2 企业级核心需求排序

| 优先级 | 需求 | 为什么重要 |
|--------|------|-----------|
| **P0** | 安全审核 | 防止恶意代码/数据泄露 |
| **P0** | 权限管理 | 控制访问和使用 |
| **P1** | 版本管理 | 生产环境稳定 |
| **P1** | 分发控制 | 确保正确使用 |
| **P2** | 智能检索 | 提升效率 |
| **P3** | 审计日志 | 问题追溯 |

### 5.3 一句话总结

> **外部 Skill 社区是"来源池"，企业 Skill Registry 是"治理平台"。企业级核心需求排序：安全审核 > 权限管理 > 版本管理 > 分发控制 > 智能检索 > 审计日志。**
