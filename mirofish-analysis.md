# MiroFish 项目分析报告

## 一、一句话总结

**这是一个"平行世界模拟器"** —— 上传现实世界的种子信息（新闻、政策、小说等），让成千上万个 AI 智能体在数字世界中自由演化，预测未来走向。

---

## 二、核心概念

### 2.1 什么是 MiroFish？

**传统预测**：
```
输入数据 → 统计模型 → 预测结果
（线性、单一、黑盒）
```

**MiroFish 预测**：
```
种子信息 → 构建平行数字世界 → 成千上万 AI 智能体自由交互 → 群体涌现 → 预测结果
（非线性、多智能体、可解释）
```

**核心创新**：
- 不是用算法预测，而是用"平行世界模拟"预测
- 每个智能体有独立人格、长期记忆、行为逻辑
- 通过群体涌现（Swarm Intelligence）预测未来

---

### 2.2 名字由来

**MiroFish** = **Miro**（镜子）+ **Fish**（鱼）

寓意：
> "鱼"在"镜子"般的数字世界中游弋，映射现实、预测未来

**中文名**：微鱼 / 微舆
- **微**：微观个体、微观舆情
- **鱼**：智能体如鱼在水中自由交互
- **舆**：舆情、舆论、民意

---

## 三、核心功能

### 3.1 你能用它做什么？

**严肃预测**：
- 📈 **金融预测**：上传财报/新闻，推演股价走势
- 📰 **舆情预测**：上传热点事件，预测舆论走向
- 🏛️ **政策推演**：上传政策草案，模拟社会反应

**趣味仿真**：
- 📖 **小说结局预测**：上传《红楼梦》前 80 回，预测失传结局
- 🎮 **脑洞推演**：如果某个历史事件改变，世界会怎样？
- 🎭 **角色对话**：与模拟世界中的任意角色对话

---

### 3.2 使用流程

```
第 1 步：上传种子材料
├── 数据分析报告
├── 新闻文章
├── 政策草案
└── 小说故事

第 2 步：描述预测需求
"预测这个政策出台后的社会反应"
"推演《红楼梦》失传的结局"

第 3 步：MiroFish 自动执行
├── 提取现实种子信息
├── 构建平行数字世界
├── 注入成千上万个智能体
├── 智能体自由交互演化
└── 生成预测报告

第 4 步：获取结果
├── 详尽的预测报告
└── 可深度交互的数字世界
```

## 四、技术架构

### 4.1 核心流程

```
┌─────────────────────────────────────────────────────────┐
│                    MiroFish 架构                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. 图谱构建                                             │
│     └→ 现实种子提取 → 个体/群体记忆注入 → GraphRAG      │
│                                                          │
│  2. 环境搭建                                             │
│     └→ 实体关系抽取 → 人设生成 → 环境配置 Agent          │
│                                                          │
│  3. 开始模拟                                             │
│     └→ 双平台并行模拟 → 自动解析需求 → 动态更新记忆      │
│                                                          │
│  4. 报告生成                                             │
│     └→ ReportAgent 与模拟后环境深度交互 → 生成报告       │
│                                                          │
│  5. 深度互动                                             │
│     └→ 与任意智能体对话 → 与 ReportAgent 对话            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

---

### 4.2 技术栈

| 组件 | 技术 | 版本要求 |
|------|------|---------|
| **前端** | Node.js + npm | 18+ |
| **后端** | Python | 3.11 - 3.12 |
| **包管理器** | uv | 最新版 |
| **LLM** | 支持 OpenAI SDK 格式 | 推荐阿里百炼 qwen-plus |
| **记忆系统** | Zep Cloud | 免费版即可 |
| **仿真引擎** | [OASIS](https://github.com/camel-ai/oasis) | CAMEL-AI 开源 |

---

### 4.3 关键组件

#### 4.3.1 智能体（Agent）

每个智能体具备：
- 🧠 **独立人格**：性格、偏好、价值观
- 📚 **长期记忆**：记住过去的经历和交互
- 🎯 **行为逻辑**：基于人格和记忆做决策
- 💬 **社交能力**：与其他智能体交互

#### 4.3.2 GraphRAG

**作用**：构建现实世界的数字映射

```
现实世界 → 实体关系抽取 → 知识图谱 → GraphRAG → 数字世界
```

**优势**：
- 不是简单的文本检索
- 理解实体之间的关系
- 支持复杂推理

#### 4.3.3 ReportAgent

**职责**：
- 与模拟后的环境深度交互
- 收集智能体行为数据
- 分析群体涌现模式
- 生成预测报告

## 六、部署方式

### 6.1 源码部署

```bash
# 1. 复制配置文件
cp .env.example .env

# 2. 编辑 .env，填入 API 密钥
# 必需：LLM_API_KEY, ZEP_API_KEY

# 3. 一键安装所有依赖
npm run setup:all

# 4. 启动服务
npm run dev

# 访问地址：
# 前端：http://localhost:3000
# 后端：http://localhost:5001
```

---

### 6.2 Docker 部署

```bash
# 1. 复制配置文件
cp .env.example .env

# 2. 启动 Docker
docker compose up -d

# 访问地址同上
```

---

### 6.3 成本估算

**LLM API**：
- 推荐：阿里百炼 qwen-plus
- 消耗：较大（成千上万个智能体交互）
- 建议：先进行<40 轮模拟测试

**Zep Cloud**：
- 免费版：每月免费额度可支撑简单使用
- 付费版：根据使用量升级

**总体成本**：
- 个人用户：免费版 + 少量 API 调用 ≈ 每月几十元
- 企业用户：根据模拟规模，可能数百到数千元

## 七、核心优势

### 7.1 vs 传统预测模型

| 维度 | 传统预测 | MiroFish |
|------|---------|----------|
| **方法** | 统计模型、机器学习 | 多智能体模拟 |
| **可解释性** | 黑盒 | 白盒（可追溯每个智能体决策） |
| **非线性** | 难以处理 | 天然支持（群体涌现） |
| **交互性** | 无 | 可与任意角色对话 |
| **趣味性** | 低 | 高（数字沙盘） |

---

### 7.2 适用场景

**✅ 适合**：
- 舆情预测和公关策略预演
- 政策出台前的社会反应模拟
- 金融市场情绪推演
- 文学创作辅助（结局预测）
- 游戏/影视剧情推演
- 教学演示（历史事件重演）

**❌ 不适合**：
- 需要精确数值预测（如股价具体数字）
- 实时性要求极高（模拟需要时间）
- 预算有限（LLM API 消耗较大）

## 十、总结

### 10.1 核心价值

**于宏观**：
> 决策者的预演实验室，让政策与公关在零风险中试错

**于微观**：
> 个人用户的创意沙盘，推演小说结局、探索脑洞，有趣好玩

**一句话**：
> 让每一个"如果"都能看见结果，让预测万物成为可能

---

### 10.2 技术意义

**创新点**：
1. **群体智能预测**：不是单一模型，而是群体涌现
2. **平行数字世界**：构建高保真模拟环境
3. **可解释性**：每个决策都可追溯
4. **深度交互**：可与模拟世界任意角色对话

**局限性**：
1. **成本**：LLM API 消耗较大
2. **速度**：模拟需要时间，非实时
3. **精度**：适合趋势预测，不适合精确数值

---

### 10.3 适合谁用？

**✅ 推荐**：
- 政府/企业决策者（政策/公关预演）

- 作家/编剧（剧情推演）
- 研究人员（社会实验模拟）
- AI 爱好者（体验多智能体技术）

**❌ 不推荐**：
- 需要精确数值预测
- 预算有限
- 需要实时结果
